설치 방법

GitLab 저장소에서 직접 설치합니다.

# 최신 버전 (main 브랜치)
uv add git+https://cbgitlab.nicecredit.co.kr/cskim/nice-cidas.git

# 특정 버전 (태그 지정)
uv add git+https://cbgitlab.nicecredit.co.kr/cskim/nice-cidas.git@v0.5.0

업그레이드하려면, 작업폴더 pyproject.toml의 nice-cidas 의존성에 박힌 rev 값을 새 태그로 바꾼 뒤 uv sync 합니다.

nice-cidas = { git = "https://cbgitlab.nicecredit.co.kr/cskim/nice-cidas.git", rev = "v0.5.0" }
uv sync
분석 안정성을 위해 각 작업폴더는 rev = "v0.5.0"처럼 버전을 고정해 설치합니다. 이 경우 uv lock --upgrade-package nice-cidas은 rev가 고정돼 있어 동작하지 않으므로, 위처럼 rev 값을 직접 바꾸고 uv sync 해야 합니다.

resolve_base_dir() v0.2.1

data의 read_csv와 nw의 read_query·read_scr·read_css가 파일을 찾을 때 쓰는 기준 경로(base_dir) 처리 규칙의 단일 진실원천입니다. 네 함수가 완전히 동일하게 동작하며, resolve_base_dir(base_dir)는 그 규칙으로 확정되는 최종 기준 폴더를 직접 돌려주는 공개 API입니다 — 노트북에서 "지금 어디를 보고 있나"를 확인할 때 사용합니다.

cs.resolve_base_dir(base_dir: str | None) → Path

두 가지 동작 — 지정 vs 자동

base_dir동작
값 지정자동 탐색을 하지 않고 그대로 사용. 상대경로면 현재 작업 폴더(CWD) 기준으로 풀림
None (기본값)아래 우선순위로 처음 "존재하는" 폴더를 자동 선택

자동 탐색 우선순위 (base_dir=None)

순위탐색 위치조건
1../data/rawCWD 폴더명이 정확히 notebooks일 때만
2../dataCWD 폴더명이 정확히 notebooks일 때만
3./data/raw항상
4./data항상
5./ (현재 폴더)항상

위에서부터 검사해 처음 존재하는 폴더를 채택합니다. 1·2순위는 CWD 폴더명이 정확히 notebooks(대소문자 무관)일 때만 후보에 추가되어, 작업폴더/notebooks에서 실행하든 작업폴더에서 바로 실행하든 동일하게 작업폴더/data/raw를 인식합니다(작업폴더 실행 시 상위의 엉뚱한 data를 잘못 읽지 않음). 5순위 ./는 항상 존재하므로 자동 탐색은 반드시 하나를 반환합니다.

파일명만 vs 경로 포함

base_dir은 파일명만 입력했을 때만 결합됩니다. 경로가 한 칸이라도 포함되면(./sub/x.gz, ../x.gz, 절대경로 등) base_dir은 무시되고 입력 경로를 그대로 씁니다. nw의 세 함수는 파일명 대신 식별자(nw_no+query_id / nw_no+score_cd / nw_no)만 줘도 base_dir 폴더 안을 훑어 파일을 찾습니다.

흔한 함정 — 노트북에서 상대경로 base_dir 작업폴더/notebooks에서 base_dir="data/raw"를 주면 작업폴더/notebooks/data/raw를 찾다가 "해당 디렉토리가 존재하지 않습니다"가 납니다. 데이터가 작업폴더/data/raw에 있다면 base_dir을 생략(자동 탐색이 이 경우를 처리)하거나 "../data/raw"·절대경로를 쓰세요. 자동 탐색은 폴더의 존재만 보고 그 안의 파일 유무는 보지 않습니다. 
import cidas as cs

# 지금 자동으로 잡히는 기준 폴더 확인
cs.resolve_base_dir(None)

# 지정값이 실제로 어디로 풀리는지 확인 (상대경로는 CWD 기준!)
cs.resolve_base_dir("../data/raw")
못 찾을 때 안내 · _LIST 파일 식별자 자동 탐색이 실패하면 base_dir을 지정하지 않은(None) 경우에 한해 위 우선순위 경로 목록을 오류 안내에 덧붙입니다. read_css의 요약항목 리스트(_LIST) 파일은 이 우선순위를 다시 타지 않고 본파일이 발견된 폴더(본파일 옆)에서 찾습니다.

cidas.sql v0.5 신규

cs.sql은 SAS PROC SQL의 대응 함수입니다. DuckDB를 백엔드로, 호출 스코프의 DataFrame 변수명을 FROM절에 그대로 쓰면 자동 인식합니다(replacement scan). Apache Arrow 제로카피를 경유하므로 pandas ↔ polars 복사 없이 결과를 반환하며, 입력 테이블은 pandas·polars가 혼재해도 됩니다.

backend 기본값은 설정 체계로 해소 backend를 명시하지 않으면 ①함수 인자 → ②세션 set_config() → ③환경변수 → ④프로젝트 설정 파일(.cidas/config.toml) → ⑤내장 기본값(pandas) 순으로 값을 찾습니다. 프로젝트 전체를 polars로 쓰려면 .cidas/config.toml에 backend = "polars" 한 줄이면 됩니다(설정 탭 참고).

import cidas as cs
import pandas as pd
import polars as pl

# 호출 스코프 변수명을 FROM에 그대로 사용 (자동 감지)
df = pd.DataFrame({'dept': ['A', 'A', 'B', 'B', 'C'], 'amt': [100, 200, 50, 300, 80]})

result = cs.sql("""
    SELECT dept, SUM(amt) AS total
    FROM df
    GROUP BY dept
    HAVING SUM(amt) > 100
    ORDER BY total DESC
""")
#   dept  total
# 0    B    350
# 1    A    300

# tables= 탈출구(동적 테이블명·테스트) + polars 반환
result_pl = cs.sql("SELECT dept, COUNT(*) AS n FROM sales GROUP BY dept",
                   tables={"sales": df}, backend="pl")

# pandas ↔ polars 혼재 입력 — DuckDB가 둘 다 스캔
pd_df = pd.DataFrame({'k': [1, 2], 'v': ['a', 'b']})
pl_df = pl.DataFrame({'k': [1, 3], 'score': [90, 80]})
out = cs.sql("SELECT pd_df.k, v, score FROM pd_df JOIN pl_df ON pd_df.k = pl_df.k", backend="pl")
FROM에 쓰는 이름은 호출 스코프의 Python 변수명 그대로입니다(공백·특수문자가 든 이름은 자동 감지 불가 → tables=로 별칭 지정). SAS CALCULATED 재참조는 DuckDB가 같은 SELECT alias 재참조를 대체로 허용하며, 안 되는 경우만 CTE(WITH)/서브쿼리로 풉니다. 자세한 사용법은 docs/sql.md를 참고하세요.

요약항목 리스트 파일 자동 구성

NW는 결과 파일과 함께 요약항목 리스트 파일을 생성합니다. 이 파일을 읽어 수천 개 컬럼명을 직접 나열하지 않고 total_cols를 자동 구성합니다.

  • 파일 대응: 결과 N001_CSALINF_OCR2.260601.gz ↔ 리스트 N001_CSALINF_OCR2.260601_LIST.gz
  • 리스트 내용: comma 구분 한 줄 — 추출포맷, 요약항목개수, 항목1, 항목2, ...
  • 구성 결과: total_cols = key_cols + 리스트항목
  • 지원 포맷: css_00009·css_90009(요약항목 99개 초과)(comma)
  • 탐색 위치: css_list 생략 시 리스트 파일은 base_dir 우선순위를 다시 타지 않고 본파일이 발견된 폴더(본파일 옆)에서 {stem}_LIST{suffix}로만 찾습니다
# 가장 간단한 사용 — 리스트 파일 자동 유도 + 기본 key_cols
df_css = cs.read_css(file_path="N001_CSALINF_OCR2.260601.gz")

# 식별자가 사업자번호인 경우 key_cols 지정
df_css = cs.read_css(
    file_path="N001_CSALINF_OCR2.260601.gz",
    key_cols=["biz_no", "base_date"]
)

# 기존 방식 (total_cols 직접 입력) — 그대로 동작
df_css = cs.read_css(total_cols=my_vars, file_path="N001_CSALINF.gz")

# dtypes로 일부 컬럼만 타입 지정 (css.toml lookup 위에 overlay, 권장)
df_css = cs.read_css(total_cols=my_vars,
    dtypes={'float64': ['V001', 'V002']}, file_path="N001_CSALINF.gz")

# NW번호 자동 탐색 + polars
df_pl = cs.read_css(nw_no="12345", backend='polars')
